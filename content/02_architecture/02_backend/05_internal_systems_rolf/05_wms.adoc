= WMS - Система управления складом

{empty} +

****
*WMS*-система управления складом (**W**arehouse **M**anagement **S**ystem) – система, предназначенная для автоматизации управления процессами склада и работы складского комплекса. +
****

== Подключение к Safe Mobile

=== Для Android

****
В браузере телефона перейдите по ссылке:

[source, html]
----
https://mail1.rolf.ru/login
----

Подтвердите, что вы доверяете сертификату и хотите перейти на сайт. Далее необходимо установить на телефон приложение Монитор, загрузка и установка начнется по нажатию на кнопку.
****

****
image::wms_1.png[title="", align=center]
****

****
После установки приложения его необходимо открыть и выбрать пункт `«Доменная учетная запись»`
Заполнить поля:
====
* *URL* сервера:
[source,html]
----
https://mail1.rolf.ru
----
* *Логин*:
[source,html]
----
ваш_логин@int.rolfcorp.ru
----
(Логин чувствителен к регистру, поэтому необходимо вводить его в том виде, в котором он у вас сделан. В компании все логины имеют вид `AAIvanov`, где первые буквы имени, отчества и фамилии большие, если вы получаете ошибку на моменте входа – напишите письмо на группу поддержки: *##UEM#* (`[.red]#uem@rolf.ru#`))
====
****

****
После входа в систему начнется процесс настройки. Предоставьте, пожалуйста, приложению все запрашиваемые разрешения. Доступ к вашим личным данным приложение не получает, оно создает отдельный рабочий профиль, в котором настраивает рабочее окружение и имеет доступ только к этому рабочему профилю.

== Общая информация о системе

****
Пользователю с ролью `«Логист»` доступны в системе несколько экранов:
====
* Назначение в ДЦ
* Автомобили
* Загрузка стока
* Уведомления
* Группа экранов `Задачи` с сортировкой всех задач в системе
====
Пользователю с ролью `«Техник»` доступны в системе 2 экрана:
====
* Все задачи
* Мои задачи
====

В списке могут быть как задачи, так и подзадачи. Подзадачи включают в себя действия, которые необходимо произвести в рамках основной задачи. +
Например, задача по Предпродажной подготовке включает в себя отправку *А/М* на сервис (подзадача `«Сервис»`) и последующую мойку (подзадача `«Мойка»`).
Слева у каждой задачи есть значок , по которому доступны действия (link:#common_wms_13_anchor[Рисунок 18, id=common_wms_13_anchor_link]):
====
* Взять в работу - С помощью этого действия можно взять в работу только одну, основную задачу.
====
При клике, в работу будет взята только задача `«Предпродажная подготовка»`. Этот вариант нужен, если подзадачи будет выполнять другой техник.
====
* Взять всё в работу – С помощью этого действия можно взять в работу основную задачу, а также все связанные с этой задачей подзадачи.
====
При клике, в работу будут взяты задача `«Предпродажная подготовка»` и *2* подзадачи `«Сервис»` и `«Мойка»`. Если весь процесс по предпродажной подготовке будет выполнять один человек, то удобнее взять всё в работу сразу.

{empty} +

== Пользовательская инструкция Техник

Все задачи и подзадачи важно синхронизировать с реальными действиями. Если авто перемещается в сервис, то необходимо у подзадачи `«Сервис»` изменить статус на `«В работе»`. Когда авто выехало из сервиса, статус у подзадачи необходимо изменить на `«Завершено»`. +
Чтобы начать работу с задачей, необходимо на экране `«Мои задачи»` нажать на значок image:properties.png[] и выбрать `«Изменить»`. Откроется окно с информацией по задаче (link:#technik_wms_1_anchor[Рисунок 36, id=technik_wms_1_anchor_link]), на которой доступны вкладки для работы. Количество и типы вкладок могут быть разные, в зависимости от типа задачи.
На вкладке `«Сменить статус»` доступна смена статуса задачи/подзадачи. Список статусов отображается поочередно, нельзя перейти из статуса `«Новая»` в статус `«Завершено»` минуя статус `«В работе»`.
