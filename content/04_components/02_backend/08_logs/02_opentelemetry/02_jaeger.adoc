== О системе Jaeger

****
*Jaeger* представляет собой распределенную систему логирования, произведенную link:https://uber.github.io/#/[*Uber Technologies*, window=_blank] с открытым исходным кодом . Он используется для мониторинга распределенных систем на основе микросервисов, в том числе:
====
* Распространение распределенного контекста
* Мониторинг распределенных транзакций
* Анализ причин
* Анализ зависимостей службы
* Оптимизация производительности
====
****

== Функциональность

****
* Модель данных на основе link:https://opentracing.io/[*OpenTracing*, window=_blank]
* Использует согласованную предварительную выборку с индивидуальной вероятностью для каждой службы/endpoint-а точки.
* Несколько встроенных серверных хранилищ: *Cassandra*, *Elasticsearch*, *in-memory*
* Внешнее хранилище, поддерживаемое через плагин *gRPC*: link:https://github.com/jaegertracing/jaeger-clickhouse[*ClickHouse*, window=_blank]
* Графики топологии системы
* Адаптивная выборка
* Конвейер обработки данных после сбора
* Мониторинг производительности служб (*SPM*)
****

====
TIP: Более подробная информация на странице link:https://www.jaegertracing.io/docs/1.46/features/[*Features*, window=_blank].
====

== Технические характеристики

****
* Бэкенд-компоненты, реализованные на *Go*
* Пользовательский интерфейс *React/Javascript*
* Поддерживаемые серверные хранилища:
** link:https://www.jaegertracing.io/docs/1.46/deployment/#cassandra[*Cassandra 3.4+*, window=_blank]
** link:https://www.jaegertracing.io/docs/1.46/deployment/#elasticsearch[*Elasticsearch 5.х 6.х 7.х*, window=_blank]
** link:https://www.jaegertracing.io/docs/1.46/deployment/#kafka[*Kafka*, window=_blank]
** Хранение в памяти
** Сертифицированные *grpc*-плагины:
*** link:https://www.jaegertracing.io/docs/1.46/deployment/#sidecar-model[*ClickHouse*, window=_blank]
****
